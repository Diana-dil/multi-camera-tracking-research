# Протокол первого исследования

## Исследовательский вопрос

Как выбор алгоритма однокамерного трекинга влияет на устойчивость локальных ID и скорость обработки видео?

## Сравниваемые варианты

1. YOLO11n + ByteTrack.
2. YOLO11n + BoT-SORT.

## Контролируемые условия

Для обоих запусков должны совпадать:
- исходное видео;
- модель детектора;
- порог confidence;
- IoU threshold;
- размер входного изображения;
- вычислительное устройство.

Меняется только трекер.

## Автоматически собираемые показатели

- число обработанных кадров;
- время обработки;
- FPS обработки;
- число уникальных локальных ID;
- количество наблюдений треков;
- длительность каждого трека;
- средняя уверенность детектора;
- длина траектории в пикселях.

## Ручная оценка

На одном и том же фрагменте вручную заполнить:
- реальное число людей;
- число смен ID;
- число фрагментированных траекторий;
- число случаев потери человека после окклюзии;
- комментарии об ошибках.

## Ограничение интерпретации

Количество уникальных треков само по себе не является метрикой качества: лишние ID могут означать фрагментацию. Поэтому автоматические показатели необходимо дополнить ручным анализом ошибок или эталонной разметкой.
